Pokemon API

I used a pokemonAPI to get data a simply retrieve it with a request in javascript. I learned from this how easy a connection can be and used a different way to show these items by using a map function.

For the duel part I've written JS code based on the API and used that to retrieve all 151 pokemons and create a matchmaking system against a random bot.

With this I know now how to use a basic API and use there data with JS.

Authentication system

I made a login system with firebase authentication and website cookies. I've learned now how to setup a firebase database and use the authentication in combination with cookies and local storage. It went very good!

Version control

LO3 – Iterative design

Blender house

The first iteration is a door of a house. The feedback was that I should expend on it.

For the second iteration I created the whole house based on a american/western house. And the feedback on that was I should spend more time on the little details to make this scene more picture perfect.

As last I added a rain pipe, which was difficult first, but I managed to create it.

Now I've learned how to use the basic tools of blender

Blender hallway

For this hallway in blender I used the basic tools of blender and the feedback was that I should start work with UV-mapping and make use of lighting

For the second itteration I went in with UV mapping for the blood piles and the walls. I now learned the really basics of uv-mapping. And the feedback for this was, that it should represent a scene, so I could make it more lively.

So I implented a creature to make the scene more lively and use window lighting, since I created some windows in the hallway.

I've learned now how to work with uv-mapping, the very basics. I also know how to place some lighting and play with the settings of it.
